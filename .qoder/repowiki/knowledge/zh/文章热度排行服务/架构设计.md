- 采用标准 Spring Boot 分层架构：Controller (`RankingController`) 暴露 REST API，Service (`RankingService`) 封装业务逻辑，VO (`HotArticleVO`) 承载响应数据。
- 核心依赖 Redis 进行高性能排行计算，通过 `RedisConfig` 显式配置 `StringRedisTemplate` 以支持 ZSET 操作（Key: `article:hot:ranking`）。
- 遵循单一职责原则，将热度增量规则（阅读+1、发布+2、评论+3、点赞+5）内聚于 Service 层，通过原子操作 `ZINCRBY` 保证并发安全。
- 入口类 `RankingApplication` 独立启动，监听端口 8083，通过 `pom.xml` 声明对 `common` 模块及 Redis Starter 的依赖。
