## 2. 启动 Docker 基础服务

先确认 Docker Desktop 已打开，并且 Docker 正在运行。

在项目根目录执行：

```powershell
docker compose up -d
docker ps
```

正常情况下，`docker ps` 中应看到以下容器：

```text
akh-mysql
akh-redis
akh-rabbitmq
```

RabbitMQ 管理页面：

```text
http://localhost:15672
```

默认账号密码：

```text
guest / guest
```

如果 Docker 命令失败，先检查命令是否写成了正确格式：

```powershell
docker compose up -d
```

不要写成：

```powershell
docker -compose up -d
```

## 3. 编译后端项目

在项目根目录执行：

```powershell
mvn clean install -DskipTests
```

编译成功时应看到：

```text

```

如果编译失败，需要先处理错误后再继续启动服务。

## 8. 建议讲解重点

验收时可以按下面顺序说明项目亮点：

1. Spring Cloud Gateway 作为统一入口，所有外部请求通过 `8080` 网关访问。
2. 用户登录后生成 JWT，网关负责鉴权并向下游服务透传用户信息。
3. 文章发布后会触发 RabbitMQ 广播事件，实现异步处理。
4. Redis ZSET 用于维护文章热度排行榜。
5. 文章详情接口经过网关限流保护。
6. AI 服务支持普通续写和 SSE 流式续写。
7. 前端只是验收演示台，核心功能都来自后端真实接口。

## 9. 常见问题处理

### 9.1 Docker 命令报错

确认命令是：

```powershell
docker compose up -d
```

如果 `docker version` 或 `docker ps` 卡住，通常是 Docker Desktop 没有正常启动，或 WSL/虚拟化环境异常。先重启 Docker Desktop，再重试。

### 9.2 Maven 编译失败

先查看错误模块和错误类。

如果是网关中 `log` 找不到符号，说明日志字段编译失败，需要确认以下两个类已经使用显式 `LoggerFactory`：

```text
gateway-service/src/main/java/com/ai/knowledgehub/gateway/ratelimit/RateLimitService.java
gateway-service/src/main/java/com/ai/knowledgehub/gateway/filter/RateLimitGlobalFilter.java
```

### 9.3 前端提示 401

说明当前没有登录，或 Token 已失效。

处理方式：

1. 在前端点击“退出”。
2. 重新点击“登录”。
3. 再操作受保护接口。

### 9.4 前端提示网络错误

优先检查：

1. `gateway-service` 是否已经启动。
2. 页面右上角网关地址是否为 `http://localhost:8080`。
3. 浏览器是否能打开 `http://localhost:8080/actuator/health`。

### 9.5 网关返回 502

通常是下游服务没有启动。

检查以下服务是否都在运行：

```text
user-service      8081
article-service   8082
ranking-service   8083
ai-service        8084
```

### 9.6 排行榜为空

排行榜依赖 Redis 热度数据。

先执行以下操作再刷新 Top10：

1. 发布文章。
2. 查看文章详情。
3. 点赞文章。
4. 发表评论。

### 9.7 RabbitMQ 页面打不开

确认容器是否运行：

```powershell
docker ps
```

确认 `akh-rabbitmq` 存在后，再打开：

```text
http://localhost:15672
```
